# 機能検討メモ

## CSVによる一括インポート

### 結論
**当面は実装しない。要望が出てから対応する。**

### 理由
- 20件以下であれば手入力（テキスト入力→Enter）で1〜2分で完了する
- CSVファイルを準備するコストと大差ない
- このアプリの用途（固定スキャン用コードの作成）は一度生成すれば再利用できる

### 実装する場合の条件
- 50件以上かつ既存リスト（Excel・DBなど）がある場合
- リストが定期更新されて再生成が必要な場合

### フォーマット案（保留）
- **案A（推奨）**: テキストファイル、1行1件、種別は画面選択に従う
- **案B**: 2列CSVヘッダーなし `QR,text` / `Barcode,text`
- **案C**: 2列CSVヘッダーあり `type,text`

## 印刷ボタン

### 結論
**当面は実装しない。要望が出てから対応する。**

### 理由
- PDF出力 → OS標準の印刷ダイアログで代替できる
- 印刷レイアウトの調整（用紙サイズ・余白・部数）は PDF 経由の方が柔軟

### 実装する場合の条件
- 「ワンクリックで印刷したい」という要望が複数あった場合
- PDF 出力を挟むことへの不満が出た場合

### 実装方針案
- `win32print` (pywin32) を使う案: 依存が重い
- `subprocess` で `mspaint /p` または `SumatraPDF /print-to-default` を呼ぶ案: 外部ツール依存
- PDF 生成後に `os.startfile(path, "print")` を呼ぶ案: **最小構成で実現可能 (推奨)**

## バーコード規格の追加

### 結論
**当面は Code128 のみ。要望が出てから追加する。**

### 理由
- Code128 は英数字・記号を扱える汎用規格で、固定スキャン用途には十分
- 規格を増やすと UI の選択肢が増え、使い勝手が下がる可能性がある

### 追加候補と用途
| 規格 | 主な用途 |
|------|----------|
| EAN-13 / JAN | 商品バーコード (数字13桁) |
| QR39 (Code39) | 工場・物流の資産管理 |
| ITF / NW-7 | 物流・宅配伝票 |

### 実装する場合の条件
- 特定規格での運用が必要という要望が出た場合
- `python-barcode` は上記規格をすでにサポートしているため、UI 追加のみで対応可能

## QRコードのカスタマイズ

### 結論
**当面は実装しない。デフォルト設定で十分なユースケースがほとんど。**

### カスタマイズ候補
- **誤り訂正レベル**: L / M / Q / H (現在は `qrcode` デフォルトの M)
  - H にすると汚れ・破損に強くなるが、QR が大きくなる
- **サイズ (box_size / border)**: 現在はデフォルト値のまま
- **前景色 / 背景色**: ロゴ入りQRなどに使われるが、スキャン精度が下がる恐れあり

### 実装する場合の条件
- ロゴ入り・カラーQRなどのデザイン用途の要望が出た場合
- `generate_qr()` のシグネチャに `error_correction`, `box_size` を追加するだけで対応可能
